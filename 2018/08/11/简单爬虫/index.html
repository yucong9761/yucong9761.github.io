简单爬虫

  • BeautifulSoup
  • python
  • requests
  • 爬虫

通过requests+BeautifulSoup进行爬虫，之前博客写过但是觉得比较粗略。通过这一段时间的实践，结合自己体会重新整理一下。

先前条件安装

python3.6安装

百度python，去官网https://www.python.org/，downloads模块下找到适合自己电脑的系统下载最新的3.6进行下载安装。使用默认设置一路下一步即可。

requests库安装

管理员权限cmd，运行pip install request，安装成功

测试一下：

BeautifulSoup库安装

win平台，管理员身份运行cmd，执行pip install beautifulsoup安装成功

爬虫实例

以豆瓣为例，首先需要想要爬取网站的url，即地址栏内网址https://www.douban.com/

BeautifulSoup库

官方文档：http://beautifulsoup.readthedocs.org/zh_CN/latest

BeautifulSoup库通过soup=BeautifulSoup(demo,"html.parser")将复杂的HTML文档转换成一个复杂的树形结构，每个结构都是python对象。包括：

  • Tag
  • NavigableString
  • BeautifulSoup
  • Comment

tag标签

tag标签：可以理解为HTML中的尖括号包含着的。eg:<a class="sister" ref="http://example.com/elsie" id="link1">Elsie</a>就是a标签。

输出结果：

通过soup加标签名轻松获取标签内容，这时查找的是soup中第一个符合要求的标签。
对于tag，有两个重要的属性：

  • name
  • attrs
    name就是标签名，对于soup对象的name即为[document]
    attrs是属性，一个字典类型

要想输出某个属性，可以

可以对属性值进行修改和删除（不是主要用途）

NavigableString

根据tag我们可以找到相应的标签内容，当我们需要标签内部文字时，用.string即可。

soup.a.string类型就是NavigableString。

BeautifulSoup

BeautifulSoup对象是一个文档全部内容，可以把它当作tag标签，即上文提到的soup

Comment

Comment对象是特殊类型的NavigableString对象，用的较少但需要注意。文档的注释部分

遍历文档树

直接子节点

.contents:将tag的子节点以列表的方式输出。

.children:返回不是一个list，但可以遍历获取所以子节点。

.descendants：对所有tag的子孙节点递归循环，需要遍历获取其中内容。与children类似。

节点内容

.string属性：如果一个标签内没有标签，则.string会返回标签里面的内容。如果一个标签内只有唯一的标签，那么.string也会返回最里面的内容。如果里面有多个标签，则结果为None。

多个内容

.strings：获取多个内容，需要遍历获取输出。

.stripped_strings：上面输出结果中包含很多空格和空行，.stripped_strings可以去除多余空白内容。

父节点

.parent

.parents全部父节点，遍历输出

兄弟节点

.next_sibling:该节点下一个兄弟节点
.previous_sibling:该节点前一个兄弟节点
不存在，返回None
.next_siblings，.previous_siblings全部兄弟节点。

前后节点

.next_element，.previous_element：不是针对兄弟节点，而是所有节点不分层次
.next_elements，.previous_elements：向前或向后迭代访问

搜索文档树

find_all()

find_all(name,attrs,recursive,text,**kwargs)搜索当前的tag的所有tag子节点。

name参数

查找所有名字为name的tag，字符串对象自动忽略
传字符串
传入字符串参数，查找与字符串完整匹配的内容。

传正则表达式
通过正则表达式的match()匹配内容，eg：找出所有以b开头的标签。表示和
应该被找到

传列表
传入列表参数，与列表中任一元素匹配的内容返回。eg：找所有a标签和br标签

传True
True可以匹配任何值，查找所有的tag，不会返回字符串节点
传方法
没有合适的过滤器，可以定义一个方法，方法只接受一个元素参数，如果方法返回True表示当前元素匹配并且被找到，如果不是返回False。
校验当前元素，包含class不包含id

keyword参数

如果一个指定名字的参数不是搜索内置的参数名,搜索时会把该参数当作指定名字tag的属性来搜索,如果包含一个名字为 id 的参数,Beautiful Soup会搜索每个tag的”id”属性。

class是关键字，所以不能直接使用，要使用class_
如果传入href参数，BeautifulSoup搜索每个tag的href属性

使用多个指定名字的参数可以同时过滤tag的多个属性
有些属性在搜索不能使用，可以通过find_all()方法的attrs参数定义一个字典参数来搜索包含特殊属性的tag

text参数

text参数可以搜索文档中的字符串内容，与name参数可选值一样，接受字符串，正则表达式，列表，True

limit参数

find_all()方法返回全部搜索结构，如果文档树很大则搜索很慢，不需要全部结果时，使用limit参数限制返回结果数量，结果数量达到limit限制，停止搜索返回结果

recursive参数

调用tag的find_all()，BeautifulSoup检索当前tag所有子孙节点，若只想搜索tag直接子节点，使用参数recursive=False

CSS选择器

我们在写 CSS 时，标签名不加任何修饰，类名前加点，id名前加 #，在这里我们也可以利用类似的方法来筛选元素，用到的方法是 soup.select()，返回类型是 list。
组合查找即和写 class 文件时，标签名与类名、id名进行的组合原理是一样的，例如查找 p 标签中，id 等于 link1的内容，二者需要用空格分开。
查找时还可以加入属性元素，属性需要用中括号括起来，注意属性和标签属于同一节点，所以中间不能加空格，否则会无法匹配到。
